# Product Metrics Documentation

## North Star Metric
**Audit Completions** - The number of completed AI tool audits per month

This metric directly measures the core value the product provides to users and serves as the primary indicator of product-market fit. Each audit completion represents a user who has found value in our tool.

## Input Metrics
1. **Daily Active Users (DAU)** - Measures engagement with the tool
2. **Conversion Rate from Audit to High Savings** - Percentage of audits showing significant savings opportunities
3. **Email Capture Rate** - Percentage of users providing contact information after seeing their audit results

## Instrumentation Plan
1. Track audit completions with unique identifiers
2. Monitor time spent on the results page
3. Track sharing activity (social shares, URL generation)
4. Monitor email capture rates
5. Track user drop-off points in the audit process

## Key Performance Indicators
1. **Audit Completion Rate**: Target >70% of users who start an audit complete it
2. **High Savings Identification**: Target 30-40% of audits show >$100/month savings
3. **Email Capture Rate**: Target >50% of high-savings users provide contact information
4. **Share Rate**: Target 20% of users share their audit results
5. **User Retention**: Target 60% of users return within 30 days

## Metric that Triggers Pivot Decisions
If the audit completion rate falls below 50% after three months of consistent user acquisition, we would need to pivot the product strategy to focus on improving the user experience and simplifying the audit process.

## Implementation Tracking
- Track metrics using analytics tools
- Monitor conversion funnels
- A/B test different UI approaches to improve completion rates
- Monitor user feedback for product improvements
